11. Memory Temperature Layers
11.1 Introduction: Why Memory Needs Temperature
ApolloGPT is designed around memory, but not all memory should be treated the same way.
Some information is immediately relevant.Some information is useful for an active project.Some information belongs in long-term storage until needed.
A memory system that retrieves everything becomes noisy.A memory system that retrieves too little becomes shallow.A memory system that treats all memory equally becomes slow, expensive, and inaccurate.
ApolloGPT solves this by using Memory Temperature Layers.
Memory temperature defines how close a memory is to active execution.
In ApolloGPT, memory is organized into three primary temperature zones:
Hot Memory → immediate working contextWarm Memory → active project / workspace contextCold Memory → long-term archive and deep retrieval

11.2 Core Concept of Memory Temperature

A piece of information may be important, but not immediately relevant.
For example:
A current user instruction is hot memory. 
A project summary is warm memory. 
A two-year-old archived chat is cold memory. 
Each may matter, but they should not be retrieved or used in the same way.
Temperature prevents the system from overloading itself with irrelevant context.
