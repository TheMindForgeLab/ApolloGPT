# ApolloGPT Project Summary

ApolloGPT is a local-first, memory-centric, object-linked, multi-agent intelligence operating system. The core equation from the source materials is:

Memory + Objects + Agents + Workflows + Models + Nodes + UI = a persistent intelligence system.

The system is designed to overcome chat-based AI limitations: fragmented conversations, weak continuity, no structured project memory, no execution layer, limited automation, and poor visibility into what the AI is doing.

## Core Architecture

The permanent loop is:

Input -> Understand -> Retrieve -> Structure Context -> Route Agent -> Route Model -> Execute -> Validate -> Store -> Improve.

ApolloGPT should treat memory as the operating substrate, not as an add-on. Models are processors. Agents are specialized workers. Workflows are executable processes. Objects are the structure that connects chats, projects, files, outputs, decisions, tasks, agents, and automations.

## Interface Direction

The intended UI has three panels:

- Left: structure, including workspaces, areas, departments, projects, chats, agents, workflows, automations, memory, files, models, and nodes.
- Center: execution, including chat, documents, workflow builder, task board, media studio, code editor, and output viewer.
- Right: intelligence state, including context packet, memory used, active agents, model/node, workflow state, logs, next actions, and tools.

## Advanced Capabilities

ApolloGPT should eventually support local models through Ollama/LM Studio/vLLM, model and LoRA routing, Stable Diffusion and ControlNet media workflows, Whisper/TTS voice workflows, n8n/webhook automation, local node compute, graph memory, vector memory, project memory, reusable procedures, outcome learning, and approval-gated external actions.
